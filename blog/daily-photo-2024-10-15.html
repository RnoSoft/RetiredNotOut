Daily Photo 15th October 2024

Today’s photo was taken while I was in Porthmadog last week. It shows a steam engine taking on coal.

This is the last daily photo for a short while, I need to work out how to properly display photos on this website.

I’ve just bought myself a Microsoft Surface Pro 9. When I looked at this website on it, I noticed that a lot of the images where a bit fuzzy, images that displayed clearly on my main computer’s monitors.

During this website’s initial development, I decided that it would have a maximum width of 1,000 pixels, and a single column. I wanted to keep it simple.

So, I scaled all images to have a 1,000-pixel width. If the actual browser width is less than 1,000 pixels, it will resize the images, they will always be smaller. If the actual browser width is greater than 1,000 pixels, the website will be centred within it, so its width remains at 1,000 pixels. This way images are never re-sized to be bigger, which results in a loss of quality. This works with the monitors on my main, as they are single density monitors.

However, the surface pro has a double density display, which means it has twice as many pixels within a given area. On the Surface Pro the width is still 1,000 pixels, however, 2,000 pixels are used to display it. This means my 1,000-pixel images need to be re-sized, to be bigger. Hence the loss of quality.

As a quick fix, I resized all the daily photos, and those on the Misty Marsden post to have a width of 2,000 pixels. That solved that problem.

However, there are displays that have a higher display density. Such as the Apple iPad, which has four times the display density. So, the images with a width of 2,000 pixels would appear fuzzy on the iPad, as they need to be re-sized bigger. I could increase the width of my images to 4,000 pixels to solve that problem. However, the image size gets a lot bigger. High density displays will need a large image width. But for smaller screens, the large image is overkill. What we need are multiple images with various widths, and therefore different sizes. The browser can then pick the best image. A small one for a mobile phone. A large one for an iPad.

I’m now in the process of figuring this out, which is why there is a pause in adding new daily photos. This series of posts explains the issue in more detail.
